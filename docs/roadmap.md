# Roadmap

This roadmap reflects the current direction of `hx` after the MVP hardening
work.

## Near Term

- cut the first tagged `0.1.x` release under the documented release policy
- keep docs aligned with metric maturity and benchmark interpretation
- improve contributor ergonomics around contract migrations and proof tiers
- add richer example adopter repositories and benchmark batteries

## Next Technical Layer

- strengthen boundary pressure from a simple cut count toward a richer weighted
  graph-cut model
- move from policy-chosen heuristic thresholds toward calibrated quantitative
  models
- formalize more static surface extraction and compatibility rules for ports

## Integration Layer

- keep Codex and Gemini MCP guidance aligned with the actual server behavior
- extend external integration testing as more transports or client behaviors
  are supported

## Evaluation Layer

- broaden the benchmark task battery
- improve baseline realism
- keep benchmark reporting descriptive until calibration work justifies stronger
  claims

## Publishability Layer

- stabilize public issue intake
- keep the release process explicit and reproducible
- prepare example adopter repositories and walkthroughs over time
- keep the public contract explicit:
  CLI names,
  MCP tool names,
  artifact schemas,
  and metric maturity labels
