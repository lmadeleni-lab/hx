# Roadmap

This roadmap reflects the current direction of `hx` after the v0.8.0 release.

## Recently Completed

- batteries-included onboarding (`hx setup`, `hx bootstrap`, `hx readiness`,
  `hx suggest`)
- governed agent loop with memory-injected system prompts
- token optimization suite (chunked reads, progressive context, surface caching,
  tool compression)
- hex lattice mathematics (percolation, holonomy, isoperimetric normalization,
  nonlinear risk scoring)

## Near Term

- cut the first tagged stable release under the documented release policy
- calibrate risk weights from observed regression data (replace heuristics)
- add richer example adopter repositories and benchmark batteries
- strengthen holonomy checks with composable transition functions

## Next Technical Layer

- weighted BFS radius expansion (edge cost from port contract complexity)
- spectral gap computation for parent group cohesion (replace edge ratio)
- full Wasserstein transport cost for boundary pressure edges
- free energy formulation for architecture potential (F = E - TS)
- exploit exact hex lattice critical exponents for scaling predictions

## Integration Layer

- keep Codex and Gemini MCP guidance aligned with server behavior
- extend external integration testing for additional transports and clients
- Claude Code native integration via `.claude/CLAUDE.md`

## Evaluation Layer

- broaden the benchmark task battery
- improve baseline realism
- keep benchmark reporting descriptive until calibration work justifies
  stronger claims

## Publishability Layer

- stabilize public issue intake
- keep the release process explicit and reproducible
- prepare example adopter repositories and walkthroughs
- keep the public contract explicit: CLI names, MCP tool names, artifact
  schemas, and metric maturity labels
